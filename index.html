📋 Microsoft Azure概要

Azureとは？

Microsoft Azureは、Microsoftが提供するクラウドコンピューティングプラットフォームです。 2010年にリリースされて以来、世界中の企業や開発者に利用されています。

主な特徴

  • グローバルリーチ: 世界60以上のリージョンで展開
  • 豊富なサービス: 200以上のクラウドサービス
  • ハイブリッドクラウド: オンプレミスとの統合が容易
  • セキュリティ: エンタープライズグレードのセキュリティ
  • コスト効率: 従量課金制で無駄なコストを削減

🌍 Azureのサービスカテゴリ

カテゴリ 説明 主要サービス
コンピューティング 仮想マシン、コンテナ、サーバーレス Virtual Machines, App Service, Functions
ストレージ データストレージと管理 Blob Storage, Files, Disk Storage
ネットワーキング ネットワーク接続とセキュリティ Virtual Network, Load Balancer, CDN

🎓 Azure基礎

Azureを始める前に知っておくべきこと

1. Azureアカウントの作成

Azureを使用するには、まずMicrosoftアカウントでAzureポータルにサインアップします：

  • 無料アカウントで12ヶ月間の無料サービス
  • $200の無料クレジット（30日間）
  • 25以上のサービスが常時無料

2. Azureポータル

Azureポータルは、ブラウザベースの統合管理コンソールです。 すべてのAzureリソースを一箇所で管理できます。

🔗 https://portal.azure.com

3. リソースグループ

Azureのリソース（VM、ストレージなど）をまとめて管理するためのコンテナです。

# Azure CLI例
az group create --name myResourceGroup --location eastus

4. サブスクリプション

Azureサービスの課金単位。企業では複数のサブスクリプションで環境を分離することが一般的です。

💻 Azureコンピューティングサービス

🖥️ Azure Virtual Machines (VM)

IaaSとして提供される仮想マシンサービス。Windows、Linux両方をサポート。

  • スケーラビリティ: 必要に応じてサイズ変更可能
  • 多様なOS: Windows Server、Ubuntu、CentOS、RHEL等
  • 豊富なサイズ: 用途に応じた最適なVM サイズ選択

VM作成の基本手順

  1. リソースグループの作成
  2. 仮想ネットワークの設定
  3. VMサイズとOSイメージの選択
  4. 認証方法の設定（SSH キーまたはパスワード）
  5. ネットワークセキュリティグループの設定

☁️ Azure App Service

PaaSとして提供されるWebアプリケーションホスティングサービス。

  • 複数言語サポート（.NET、Java、Node.js、Python、PHP）
  • 自動スケーリング機能
  • CI/CDパイプライン統合
  • カスタムドメインとSSL証明書

⚡ Azure Functions

サーバーレスコンピューティングサービス。イベント駆動型の処理に最適。

  • 実行時間分のみ課金
  • 多様なトリガー（HTTP、Timer、Blob、Queue等）
  • 自動スケーリング
  • 複数言語サポート

🐳 Azure Container Services

サービス 説明 用途
Container Instances シンプルなコンテナ実行 単一コンテナ、バッチ処理
Kubernetes Service (AKS) マネージドKubernetes マイクロサービス、オーケストレーション
Container Apps サーバーレスコンテナ モダンアプリ、API、背景処理

📦 Azureストレージサービス

🗄️ Azure Storage Account

Azureストレージの基盤となるアカウント。複数のストレージサービスを包含します。

ストレージの種類

  • Blob Storage: オブジェクトストレージ（画像、動画、ログファイル等）
  • File Storage: 共有ファイルシステム（SMBプロトコル）
  • Queue Storage: メッセージキューサービス
  • Table Storage: NoSQLキーバリューストア

🔄 冗長性オプション

種類 略称 説明 可用性
ローカル冗長ストレージ LRS 同一データセンター内で3回複製 99.999999999% (11 9's)
ゾーン冗長ストレージ ZRS 3つの可用性ゾーンに複製 99.9999999999% (12 9's)
地理冗長ストレージ GRS セカンダリリージョンにも複製 99.99999999999999% (16 9's)

🗃️ Azure データベースサービス

Azure SQL Database

フルマネージドなSQL Serverデータベースサービス。

  • 自動バックアップとポイントインタイム復旧
  • 自動チューニングとスケーリング
  • 高可用性（99.99%のSLA）
  • Advanced Threat Protection

Azure Cosmos DB

グローバル分散型NoSQLデータベース。

  • 複数のAPIサポート（SQL、MongoDB、Cassandra、Table、Gremlin）
  • ミリ秒単位の低レイテンシ
  • グローバル分散と自動フェイルオーバー
  • 5つの整合性レベル

🌐 Azureネットワーキング

🔗 Azure Virtual Network (VNet)

Azureのプライベートネットワーク。オンプレミスネットワークのようにリソースを接続します。

VNetの主要機能

  • サブネット: ネットワークセグメンテーション
  • ネットワークセキュリティグループ: ファイアウォール機能
  • ルートテーブル: カスタムルーティング
  • VNet Peering: VNet間の接続

🔒 ネットワークセキュリティ

Network Security Groups (NSG)

ネットワークトラフィックをフィルタリングするセキュリティルール集合。

# NSGルールの例
Priority: 100
Name: AllowHTTP
Protocol: TCP
Source: Any
Destination: Any
Port: 80
Action: Allow

⚖️ Azure Load Balancer

高可用性とスケーラビリティを提供するロードバランサー。

種類

  • Basic Load Balancer: 基本的な負荷分散
  • Standard Load Balancer: 高度な機能と可用性
  • Application Gateway: レイヤー7ロードバランサー（WAF機能付き）

🚀 Azure CDN

コンテンツ配信ネットワーク。世界中のエッジサーバーからコンテンツを高速配信。

  • 低レイテンシでのコンテンツ配信
  • 帯域幅コストの削減
  • 動的サイト高速化
  • DDoS攻撃からの保護

🔐 Azureセキュリティ

🛡️ Azure Active Directory (Azure AD)

クラウドベースのIDおよびアクセス管理サービス。

主要機能

  • シングルサインオン (SSO): 一度のサインインで複数アプリへアクセス
  • 多要素認証 (MFA): 追加のセキュリティレイヤー
  • 条件付きアクセス: リスクベースのアクセス制御
  • Identity Protection: 異常なサインイン検出

🔑 RBAC (Role-Based Access Control)

ロールベースでAzureリソースへのアクセスを制御。

組み込みロール 権限 用途
Owner フルアクセス リソース管理者
Contributor 読み書き（アクセス管理除く） 開発者
Reader 読み取り専用 監査者

🔒 Azure Key Vault

シークレット、キー、証明書を安全に保管・管理するサービス。

  • ハードウェアセキュリティモジュール（HSM）による保護
  • アクセス ポリシーによる細かい制御
  • 監査ログとモニタリング
  • アプリケーションとの統合

🛡️ Azure Security Center

統合セキュリティ管理システム。脅威の検出と対応を支援。

  • セキュリティ スコアによる状況把握
  • 推奨事項による設定改善
  • 脅威の検出とアラート
  • コンプライアンス管理

📊 Azure監視とログ

📈 Azure Monitor

Azureリソースの包括的な監視ソリューション。

主要コンポーネント

  • メトリクス: 数値データの時系列コレクション
  • ログ: イベントデータとテキストデータ
  • アラート: 条件に基づく通知
  • ダッシュボード: 視覚的なデータ表示

📋 Log Analytics

ログデータを収集、分析するためのサービス。

KQLクエリ例

// 過去24時間のエラーログを検索
AppServiceHTTPLogs
| where TimeGenerated > ago(24h)
| where ScStatus >= 400
| summarize count() by ScStatus, bin(TimeGenerated, 1h)
| render timechart

🔔 アラートとアクション

メトリクスやログに基づいてアラートを設定し、自動的にアクションを実行。

アラートタイプ

  • メトリックアラート: CPU使用率、メモリ使用量等
  • ログ検索アラート: 特定のログパターン検出
  • アクティビティログアラート: Azureリソースの変更

🎯 Application Insights

アプリケーションのパフォーマンス管理（APM）サービス。

  • リアルタイムのパフォーマンス監視
  • 依存関係の可視化
  • ユーザー行動分析
  • 障害の自動検出

🚀 Azure DevOps

⚙️ Azure DevOps Services

開発チーム向けの協業ツールセット。プロジェクト管理からデプロイまでをカバー。

主要サービス

サービス 機能 用途
Azure Boards 作業項目追跡 プロジェクト管理、バックログ管理
Azure Repos Git リポジトリ ソースコード管理、バージョン管理
Azure Pipelines CI/CD ビルド、テスト、デプロイの自動化
Azure Artifacts パッケージ管理 NuGet、npm、Mavenパッケージ

🔄 CI/CDパイプライン

継続的インテグレーション・継続的デプロイメントの実装。

パイプラインの構成例

# azure-pipelines.yml
trigger:
- main

pool:
  vmImage: 'ubuntu-latest'

steps:
- task: UseDotNet@2
  inputs:
    version: '6.0.x'

- script: dotnet build --configuration Release
  displayName: 'Build application'

- script: dotnet test --configuration Release
  displayName: 'Run tests'

- task: DotNetCoreCLI@2
  inputs:
    command: 'publish'
    publishWebProjects: true
    arguments: '--configuration Release --output $(Build.ArtifactStagingDirectory)'
